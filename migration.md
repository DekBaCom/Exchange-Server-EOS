# Migration Guide

Migrating away from an End-of-Support (EOS) Exchange Server is critical for security and compliance. Here are the primary paths:

## 1. Moving to Microsoft 365 (Exchange Online)

This is the most common and recommended path.

### Benefits:
* **Always Up-to-Date:** No more patching or major upgrades.
* **Security:** Built-in protection against threats.
* **Cost:** Shift from CapEx to OpEx.
* **Features:** Access to Teams, SharePoint, and more.

### Migration Methods:
* **Cutover Migration:** Best for small organizations (< 150 mailboxes).
* **Staged Migration:** For older versions like Exchange 2003 or 2007.
* **Hybrid Migration:** For larger organizations needing a long-term coexistence or a gradual move.

## 2. Upgrading to Exchange Server SE

If you must remain on-premises, you should move to **Exchange Server Subscription Edition (Exchange SE)**.

### Key Considerations:
* **Subscription Model:** Requires an active subscription.
* **Modern Hardware:** Ensure your infrastructure meets the new requirements.
* **Interim Steps:** You might need to move to Exchange 2019 first before moving to Exchange SE.
